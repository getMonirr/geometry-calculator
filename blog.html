Questions and Answers

1. What is DOM and what is the purpose of DOM?

DOM stands for Document Object Model. It is a platform and language-natural interface. The purpose of DOM is to allow programmers to add programs and scripts to dynamically access and change document structure, content, and style. There are three types of DOM:

  • Core DOM
  • XML DOM
  • HTML DOM
We only discuss HTML DOM.

What is the HTML DOM?

The HTML DOM is a standard object model and programming interface for HML documents. It treats all HTML Elements as Objects. HTML DOM views an HTML document as a tree of nodes. Each node represents an HTML element. We already know that it is a programming interface so we can access all objects and it allows us to create, change or remove an element from the document. also, we can add so many events for each node or element like 'click', 'mouseEnter', 'mouseMove', etc.

2. How will you select HTML elements using DOM? Name the DOM methods.

There are five different methods for selecting an element in the HTML document. all methods are self-explained by their name.

  1. getElementById()
    • you can select only one id at a time.
  2. getElementsByClassName()
    • you can get all classes by their names that match and it gives us an HTMLCollection.
  3. getElementsByTagName()
    • you can get all classes by their tags names that match and it gives us an HTMLCollection.
  4. querySelector()
    • it selects an HTML element like a CSS selector.
  5. querySelectorAll()
    • it selects all HTML elements that match and it gives us a NodeList.

3. What is event bubble?

Events bubbling is a concept of DOM. It describes how the browser handles events. when an element receives an event, and that event bubbles up (propagated) to its parent elements in the DOM tree until it gets to the root element. this is a default behavior of the browser. it is possible to prevent that? yes of course. if you can't stop bubbling it's running until getting the root element. you can stop it by just calling the sotpPropagration() methods. after applying the method event bubbling stops

4. What is a callback function and why will you use it?

In javascript, everything is an object. when we write a function, we can pass an object as a parameter. technically in javascript functions are also an object. so definitely we can pass a function as a parameter. it's called the call-back function. that means when we passed a function into a function as a parameter, which is called or invoked inside the function, and completes some task or action.

it's also called asynchronous programming. in JavaScript run code sequentially in top-down order. it's the default behavior. when we need to do something and then run other cases and it's not sequentially so what do? the solution to the problem is we have to use the callback function to do the task in a completely asynchronous way not sequentially. that's why we use the callback function. it's very important for asynchronous programming.
